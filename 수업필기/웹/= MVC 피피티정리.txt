[MVC]
& JSP의 웹 에플리케이션 모델
(1) MODEL1 : 비즈니스 로직과 화면처리를 모두 JSP로 처리	: X
(2) MODEL2 : 비즈니스로직과 화면처리를 분리		: O
=> MODEL2를 사용한다

& model1과 model2의 차이
=> controller을 jsp로 만드냐, 서블릿으로 만드냐의 차이
=> model2는 controller과 자바빈을 백엔드로, jsp를 프론트영역으로 나눔으로서
효율적인 웹 관리가 가능해졌다

1. MVC 패턴(MODEL2 방식)
- 정의 : 화면처리, 비즈니스로직처리, 요청처리를 분리해 프로그램을 개발하는
디자인패턴
- 특징 : 여러 jsp파일을 이용하여, 결과에 따른 페이지 이동이 잦다

- 구성요소
(1) view		
- 요청화면과 응답화면
- 화면처리
- jsp파일(프론트영역)

=> 요청 
<= 응답

(2) Controller(servlet)
- 형태 : servlet	
- 기능 : 요청 인식, 요청을 처리할 model 실행, 응답할 view로 이동

=> 적잘한 model(자바 클래스) 호출
<= 처리결과 / 처리결과를 보여줄 뷰이름 전달(jsp)

(3) Service(java)		
- 형태 : java클래스
- 역할 : 비즈니스 로직 처리
- 기능 : 검색기능, 추가기능, 수정기능, 삭제기능

=> repository 호출
<= 결과 반환

(4) Repository(java 싱글턴)
=> db에 접근(dao)
=> 쿼리 실행
<= 쿼리결과 반환
<= db처리 결과를 받아서 전달

(5) db
