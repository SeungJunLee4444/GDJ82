session : 서버의 영역에 저장, 브라우저가 꺼질때까지 유지
- 웹 서버에 저장되는 쿠키(세션쿠키, 쿠키목록의 sessionId를 말함)
cookie : 클라이언트에게 저장, 브라우저가 꺼지든 말든, 정해진 지속시간만큼 유지됨


# session
* jsessionId : 웹 서버에 저장되는 쿠키(=세션쿠키)
- 쿠키보다 보안이 좋음 : 브라우저를 닫거나, 서버에서 세션을 삭제했을 때만 삭제된다
- 각 클라이언트에게 고유의 sessionId를 부여

* 쿠키와 세션은 비슷한 역할을 하며, 세션도 결국 쿠키를 사용하기 때문이다
* 쿠키는 정보가 서버에 있어 속도가 느리고, 쿠키는 빠르다
* 세션도 만료기간을 정할 수 있음



* session은 브라우저에 한개의 세션쿠키를 지니고 있으며, cookie보다 보안이 좋다
