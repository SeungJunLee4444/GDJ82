[5] DML

1. DML
=> 테이블 내 ROW값의 관리를 위한 쿼리문
=> INSERT, UPDATE, DELETE가 있으며, COMMIT을 같이 써줘야한다				& 저장
1) INSERT 
-------------------------------------------------------------------------------
=> INSERT INTO 테이블명(칼럼1, 칼럼2, 칼럼3) VALUES(값1,값2,값3);
-------------------------------------------------------------------------------

* 주의사항
(1) NOT NULL 주의
=> 수정하려는 칼럼 외에, NOT NULL 제약이 걸린 칼럼은 꼭 같이 입력해줘야 한다		& INSERT 주의사항
ex)
-------------------------------------------------------------------------------
INSERT INTO DEPARTMENT(DEPT_NO, DEPT_NAME) VALUES(5,'개발부');
-------------------------------------------------------------------------------
=> INSERT에 포함되지 않은 LOCATION 칼럼이 NOT NULL이라면, 꼭 포함해서 입력해야한다

(2) PK, UNIQUE 중복불가
------------------------------------------------------------------------------------------------
ex) INSERT INTO DEPARTMENT(DEPT_NO, DEPT_NAME, LOCATION) VALUES(4,'개발부', '인천');
------------------------------------------------------------------------------------------------
=> DEPT_NO는 PK칼럼으로, 해당 테이블에 4라는 값이 이미 존재하기 때문에, 오류발생



(3) 용량초과
ex) 
------------------------------------------------------------------------------------------------------------------
INSERT INTO DEPARTMENT(DEPT_NO, DEPT_NAME, LOCATION) VALUES(5,'개발부', '부에노스아이레스');
------------------------------------------------------------------------------------------------------------------
=> LOCATION 칼럼의 용량이 15바이트로, 용량초과로 오류


2) UPDATE
-----------------------------------------------------------------------------------
=> UPDATE 테이블명 SET 칼럼1 = 값1, 칼럼2 = 값2, 칼럼3 = 값3);
-----------------------------------------------------------------------------------

3) DELETE 
----------------------------------------------------------
=> DELETE FROM 테이블명 WHERE 칼럼1 = 값1;
----------------------------------------------------------

2. 트랜잭션
=> 여러 DML을 하나의 작업단위로 처리하는 것을 말한다

1) TCL언어
COMMIT;		: 저장하고 트랜잭션 종료
ROLLBACK;	: 수정 취소, 트랜잭션 종료
