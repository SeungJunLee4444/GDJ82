[AJAX]
& MVC와 AJAX 방식의 차이
- MVC는 페이지 이동이 있는 통신
=> 포워드, 리다이렉트, COMMON(actionforward) 사용 필요
=> 다수의 jsp 파일

* ajax를 다룰려면 제이쿼리를 같이 다룰줄 알아야한다

- AJAX는 페이지 이동이 없는 통신
=> VOID 타입의 반환통신
=> 포워드, 리다이렉트가 없음
=> 하나의 JSP 파일에서 페이지 변화없이 데이터를 주고받는 방식

- 특징
1) 서비스 : jsp가 하나 뿐이기 때문에 경로방식을 생성하지 않으며,

- 점검
post man을 이용해 데이터가 제대로 전송되었는지 확인 가능하다
- 조건 : 톰캣이 켜진 상태여야함

- reqeust.getContextPath() : 프로젝트명(컨텍스트패스)를 가져온다
=> request 내부객체를 이용한 메서드 호출

- request.getRequestURI() : 프로젝트 + 파일경로까지 가져온다

- Mine-type 
- 용도 : 서버에서 클라이언트에 데이터 응답시 어떤 종류의 데이터를 전송할지 설정
- 정의 : 톰켓 컨테이너에서 제공하는 여러가지 전송데이터 종류들이다
- 종류 
(1) text/plain 	: text
(2) application/xml : xml
(3) application/json : 제이슨
response.setContentType("application/json; charset=UTF-8")

${}와 #{}의 차이
${} : 해당 값에 따라 파라미터의 자료형이 변경된다
=> 사용자의 입력 전달에는 사용x
=> 테이블, 컬럼명을 파라미터로 전달하고 싶을 때 사용
#{} : 파라미터가 String 형태로 들어온다, 값은 자동적으로 ''를 가지게됨
=> 사용자 입력전달에 사용o
